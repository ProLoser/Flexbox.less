The Most Important Things You Need to Know About Flexbox

  1. Flex Containers: A flex container is an element whose display setting has been set to display: flex or display:inline-flex. A flex container interacts with its siblings and with its children. A flex container does not directly interact with its parent container unless that parent container is also a flex container.
  2. Inline-Flex: An inline-flexed item that has a position: absolute or any float: {value} specified is computed to be display: flex instead of display: inline-flex
  3. Flex Items: A flex item is an element that is a direct descendant of an element whose display value is set to display: flex or display: inline-flex.
    1. Anonymous Flex Items: All direct descendants of a flexed element are flex items, through inheritance. You do not need to do anything to a direct descendant element to make it a flex item. It simply is.
      Second generation descendants, however, are NOT flex items. This is an incredibly important distinction, as you start nesting HTML elements and getting unexpected behavior.
    2. Explicit Flex Items: There is no CSS rule to indicate an element is a flex item. Therefore, in order for an element to be a flex item, its parent MUST be a flex container.
    3. float: {value} and clear: {value} do not work on flex items.
    4. vertical-align: {value} does not work on a flex item.
    5. Empty Flex Items: A flex item that contains only white space is not rendered (as if it were display: none).

Flex Container Rules

  • display: flex
  • display: inline-flex
  • flex-direction: {row | column | row-reverse | column-reverse}
  • flex-wrap: {wrap | nowrap | wrap-reverse}
  • align-items: {flex-start | flex-end | center | baseline | stretch}
  • align-content: {flex-start | flex-end | space-between | space-around | center | stretch}
  • justify-content: {flex-start | flex-end | space-between | space-around | center}

Flex Container Shorthand Rules

  • flex-flow: {flex-direction | flex-wrap}

Flex Item Rules

  • order: {integer}
  • align-self: {auto | flex-start | flex-end | center | baseline | stretch}
  • flex-grow: {number}
  • flex-shrink: {number}
  • flex-basis: {number | % em pt rem}

Flex Item Shorthand Rules

  • flex: { none | initial | inherit} | { flex-grow} {flex-shrink} {flex-basis}

3. Flex Container | Flex-Direction: Column

Children With align-self Rules

The important difference here is that the Flex Child Child div elements are still acting under display: block rules--i.e. 100% of the width of their containing element.

An interesting thing to note, as well, is that the HTML for Flex Children 3, 4, and 5 are identical in scenarios 2 and 3. Because we have changed the orientation of the container, we have also changed the cross axis from vertical to horizontal, and thus the align-self rules, which are identical in both instances, behave differently.

Flex Child 2 has been given the same children as Flex Child 1 (identical markup in the HTML). Flex Child 2 has then been given display: flex and flex-direction: row, making it a flex container, and changing the layout behavior of the children.
